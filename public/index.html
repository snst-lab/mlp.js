mlp.js
Introduction
This is an HTML parser that extends HTML syntax and makes your Front-end web developments more efficient.
Features
You can modularize html,css,and javascript codes.
Code splitting and modularization make your web development highly scalable.
This library parses html at runtime. You do not need to transpile the code.
You can use it by reading only one js file. There is no dependency library.
Community
Mlp.js is a public beta. If you like, join the community and cooperate in improving the source.
Mlp.js is provided with the LGPL license. Please commit, pull or clone as necessary.
Please obtain the invitation code from the icon above and join our Discord.
Custom Elements
Mlp.js provides several custom elements. It is useful for registration of styles or events to dom, code splitting or calling preloader, etc.
<flex-module>
"Flex-module" is a custom tag that applies style and register events to dom. At loading, the content of this tag automatically deleted after it is expanded into a tag with "use" attribute.

Keys for Javascript
KeyDescriptionJavascript notation
@onloadThis key specifies the processing to be executed when loading dom contents is completed.document.addEventListener('DOMContentLoaded',function(){ ... },false);
@switchEach time you click on elements using flex-module, the process specified by #1:,#2:,.. is executed in order. -
@hold [time]This key specifies the processing to be executed consecutively while holding down elements using flex-module. -
@swipe [top|right|bottom|left]This key specifies the processing to be executed when swiping elements using flex-module. -
@inview [threshold]This key specifies the processing to be executed when elements using flex-module enters the screen. -
@outview [threshold]This key specifies the processing to be executed when elements using flex-module exits the screen. -
@interval [time]This key specifies the processing to be executed periodically for elements using flex-module. setInterval(function(){ ... }, [time]);
@[Event Name]This key specifies an event listener for the elements using flex-module. click, mousedown, scroll, keyup, ... etc. Events can be enabled by specifying them in the configuration file "mlp.json".DOM.addEventListener('[Event Name]',function(){ ... });
@[Flex ID]_[Method Name]Any name following the flex id and underscore will be recognized as a custom method in grobal scope. The 1st,2nd,.. arguments of the custom method can be specified by the "#ARG0","#ARG1",.. notation in the value.window.[Flex ID]_[Method Name] = function(){ ... };
In the value of the key for javascript above, the CSS selector of its own element can be specified by "THIS" notation.

@scriptThis key generates script tag includes this value.(One flex id generates only one script tag.)<script> ... </script>

Member Functions
mlp.on (String triggerEvent)
"Mlp.on ()" is a function that triggers a javascript event. The first argument contains a character string indicating an arbitrary event name. Use it together with the "on" attribute. This is synonymous with "document.trigger ()" in jQuery.

Configure
mlp.json
You can customize the settings by editing "mlp.json". "Mlp.json" should be placed in the same directory as html file which read "mlp.js" such as "index.html".
mlp.json
PropertyValue
domSpecify the tag type to be parsed by mlp.js with csv.
eventSpecify javascript events that can be specified as keys of flex-module with csv.
screenSpecify the value that can be registered in the "screen" attribute by javascript conditional expression.
cookieIf this value is set to "enabled", cookies are not automatically deleted when loading the page or ajax request. The default value is 'disabled'.
If "cookie" property is set to "enabled", the risk of XSS increases. Please also take necessary security measures on the server side.
